## Bài 1: Ghép đôi

### Tóm tắt đề bài

Cho hai mảng $A$ gồm $n$ phần tử và mảng $B$ gồm $m$ phần tử. Hãy đếm số cặp $(i, j)$ $(1 \le i \le n, 1 \le j \le m)$ sao cho $A_i = B_j$. Hai cặp $(i_1, j_1)$ và $(i_2, j_2)$ được coi là khác nhau khi $i_1 \neq i_2$ hoặc $j_1 \neq j_2$.

#### Ràng buộc:

- $1 \le n, m \le 10^5$
- $0 \le A_i \le 10^9 \ \forall i \in \{1, 2, \dots, n\}$
- $0 \le B_i \le 10^9 \ \forall i \in \{1, 2, \dots, m\}$

#### Subtasks:

- **Subtask 1 (80\%):** $n, m \le 1000$
- **Subtask 2 (20\%):** Không có ràng buộc gì thêm.

### Lời giải

#### Subtask 1:

Ta chỉ cần duyệt qua mọi chỉ số $(i, j)$ và đếm số cặp thỏa mãn yêu cầu.

**Độ phức tạp:** $O(n \cdot m)$

#### Subtask 2:

Ta sẽ tạo hai mảng đếm $cntA$ và $cntB$. Khi đó, ta sẽ duyệt qua mọi phần tử $x$ của mảng $A$ và cập nhật $cntA_x \rightarrow cntA_x + 1$. Ta làm tương tự với mọi phần tử $x$ của mảng $B$ và cập nhật $cntB_x \rightarrow cntB_x + 1$. Khi đó, với mọi phần tử $i$ trong $cntA$ thì số cặp thỏa mãn tương ứng với giá trị này sẽ là $cntA_i \cdot cntB_i$.

Từ đó, kết quả của bài toán này sẽ là tổng của các $cntA_i \cdot cntB_i$ với mọi phần tử $i$ trong $cntA$.

**Độ phức tạp:** $O(n + m)$

```cpp
    ios_base::sync_with_stdio(false);

    }

    }
    for (int i = 1; i <= m; i++) {

    }
    ll ans = 0;

    }
    cout << ans;
    return 0;
```

## Bài 2: Nguyên tử khối

### Tóm tắt đề bài

Cho một công thức hóa học dưới dạng xâu ký tự $S$ chỉ gồm ba nguyên tố Carbon (C), Hydrogen (H) và Oxygen (O) với nguyên tử khối lần lượt là 12, 1 và 16; và chữ số trong khoảng $[2, 9]$. Hãy tính phân tử khối của công thức đó.

#### Ràng buộc:

- $|S| \le 10^6$.
- Xâu kí tự $S$ chỉ gồm các ký tự ‘C’, ‘H’, ‘O’, ‘(’, ‘)’ và chữ số trong khoảng $[2, 9]$.

#### Subtasks:

- **Subtask 1 (60\%):** Xâu ký tự $S$ không gồm hai ký tự ‘(’ và ‘)’.
- **Subtask 2 (20\%):** Xâu ký tự $S$ gồm một ký tự ‘(’ và ‘)’.
- **Subtask 3 (20\%):** Không có ràng buộc gì thêm.

### Lời giải

#### Subtask 1:

Chúng ta chỉ cần duyệt qua các ký tự trong xâu $S$, nếu ký tự đó là một trong ba ký tự ‘C’, ‘H’, ‘O’ thì ta sẽ duyệt đằng sau để lấy hệ số của nó (nếu có). Sau đó, chúng ta cộng các kết quả lại với nhau.

**Độ phức tạp:** $O(|S|)$

#### Subtask 2:

Ta sẽ tìm vị trí của hai dấu ngoặc ‘(’ và ‘)’. Gọi $L$ là vị trí của dấu ‘(’, $R$ là vị trí của dấu ‘)’.

Gọi $m_l$ là phân tử khối của đoạn trước ngoặc hay là đoạn $S_{0 \dots L - 1}$, $m_m$ là phân tử khối của đoạn trong dấu ngoặc hay là đoạn $S_{L + 1 \dots R - 1}$, $m_r$ là phân tử khối của đoạn sau ngoặc.

Khi đó, ta sẽ tính $m_l$ như subtask 1. Khi ta gặp đến ký tự ‘(’ thì ta sẽ bắt đầu tính đoạn bên trong ngoặc, và khi ta gặp ký tự ‘)’ thì ta sẽ đọc hệ số ở đằng sau (nếu có).

Sau đó, ta sẽ tính $m_r$ là đoạn còn lại của xâu $S$, ta làm như subtask 1.

Đáp án của bài toán này sẽ là $m_l + m_m + m_r$.

**Độ phức tạp:** $O(|S|)$

#### Subtask 3:

Ta sẽ dùng stack để lưu trữ lại những kết quả đã được tính. Cụ thể hơn, ta sẽ làm như sau:

- Khi duyệt các kí tự trong xâu $S$, nếu kí tự đó là một trong ba kí tự ‘C’, ‘H’, ‘O’ thì làm như subtask 1 và đẩy khối lượng vào stack.
- Khi gặp kí tự ‘(’ thì ta sẽ đẩy vào stack một giá trị đặc biệt để đánh dấu biểu thức sau là một nhóm.
- Ngược lại, khi gặp dấu ‘)’, ta sẽ lấy toàn bộ giá trị trong ngoặc bằng cách cộng dồn các phần tử trong stack cho đến khi gặp giá trị đặc biệt, loại bỏ giá trị đó khỏi stack, và đọc hệ số ở đằng sau. Khi đó, ta sẽ thực hiện phép nhân và đẩy lại kết quả đó vào stack.
- Sau khi đã duyệt hết chuỗi, đáp án sẽ là tổng các số đã được lưu trữ trong stack.

**Độ phức tạp:** $O(|S|)$

```cpp
#include <bits/stdc++.h>
#define ll long long
#define int128 __int128
#define ld long double
#define st string

using namespace std;

ll f(char c) {
    if (c == 'C') {
        return 12;
    }
    if (c == 'H') {
        return 1;
    }
    return 16;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);
    if (fopen("NTK.inp", "r")) {
        freopen("NTK.inp", "r", stdin);
        freopen("NTK.out", "w", stdout);
    }
    st s;
    cin >> s;
    stack <ll> a;
    for (int i = 0; i < s.size(); i++) {
        if (s[i] == 'C' || s[i] == 'H' || s[i] == 'O') {
            ll res = 0;
            int j = i + 1;
            while (j < s.size() && isdigit(s[j])) {
                res *= 10;
                res += s[j] - '0';
                j++;
            }
            if (res == 0) {
                res = 1;
            }
            a.push(res * f(s[i]));
            i = j - 1;
        }
        if (s[i] == '(') {
            a.push(-67);
        }
        if (s[i] == ')') {
            ll ss = 0;
            while (!a.empty() && a.top() != -67) {
                ss += a.top();
                a.pop();
            }
            a.pop();
            int j = i + 1;
            ll res = 0;
            while (j < s.size() && isdigit(s[j])) {
                res *= 10;
                res += s[j] - '0';
                j++;
            }
            if (res == 0) {
                res = 1;
            }
            a.push(ss * res);
            i = j - 1;
        }
    }
    ll ans = 0;
    while (!a.empty()) {
        ans += a.top();
        a.pop();
    }
    cout << ans;
    return 0;
}
```

## Bài 3: Lễ hội

### Tóm tắt đề bài

Cho $n$ cái cột đèn được đánh số từ 1 đến $n$. Trong đó, có $t$ cột đèn đã bị đổ, được đánh dấu bằng mảng $S$. Hãy tìm số lượng cột đèn cần sửa chữa ít nhất sao cho có ít nhất $k$ cái cột đèn còn nguyên vẹn liên tiếp.

#### Ràng buộc:

- $1 \le t, k \le n \le 10^5$
- $1 \le S_i \le n \ \forall i \in \{1, 2, \dots, t\}$

#### Subtasks:

- **Subtask 1 (50\%):** $n \le 10$
- **Subtask 2 (50\%):** Không có ràng buộc gì thêm.

### Lời giải

#### Cách 1:

Ta sẽ tạo một mảng $A$ gồm $n$ phần tử để lưu lại trạng thái của mỗi cột đèn, với $A_i = 1$ nếu đèn $i$ bị đổ và $A_i = 0$ nếu đèn $i$ không bị đổ $(1 \le i \le n)$.

Khi đó, ta có thể áp dụng kỹ thuật cửa sổ trượt để giải bài toán này. Cụ thể:

- Ta sẽ tính số lượng đèn hỏng trong đoạn $[1, k]$. Gọi đáp số này là $cnt$.
- Khi đó, ta sẽ khởi tạo $ans = cnt$.
- Với mọi $i$ từ 2 đến $n - k + 1$, ta sẽ xét đoạn $[i, i + k - 1]$ và cập nhật $cnt \rightarrow cnt - A[i - 1]$ và $cnt \rightarrow cnt + A[i + k - 1]$. Với mỗi lần lặp đó, ta sẽ cập nhật $ans \rightarrow \min(ans, cnt)$.
- Sau khi đã duyệt xong, đáp án của bài toán là $ans$.

**Độ phức tạp:** $O(n)$

<details>
<summary>Code tham khảo</summary>

```cpp
/**
*    author:  ilovedeque
 *    created: 24.04.2026 15:01:16
**/

#include <bits/stdc++.h>
#define ll long long
#define int128 __int128
#define ld long double
#define st string

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);
    if (fopen("FES.inp", "r")) {
        freopen("FES.inp", "r", stdin);
        freopen("FES.out", "w", stdout);
    }
    int n, k, t;
    cin >> n >> k >> t;
    vector <bool> a(n + 1, 0);
    while (t--) {
        int s;
        cin >> s;
        a[s] = 1;
    }
    int cnt = 0;
    for (int i = 1; i <= k; i++) {
        cnt += a[i];
    }
    int ans = cnt;
    for (int i = 2; i <= n - k + 1; i++) {
        cnt -= a[i - 1];
        cnt += a[i + k - 1];
        ans = min(ans, cnt);
    }
    cout << ans;
    return 0;

```
</details>

#### Cách 2:

**Nhận xét:** Giả sử ta có thể sửa $m$ cột đèn để tạo được một đoạn gồm ít nhất $k$ cột đèn còn nguyên vẹn liên tiếp, suy ra với mọi $m' > m$ ta cũng có thể thực hiện được điều đó. Ngược lại, nếu ta không thể sửa $m$ cột đèn để tạo được một đoạn gồm ít nhất $k$ cột đèn còn nguyên vẹn liên tiếp, suy ra với mọi $m' < m$ ta không thể thực hiện điều đó.

Vì thế, ta có thể áp dụng kỹ thuật tìm kiếm nhị phân cho số lượng cột đèn cần được sửa.

Cụ thể, giả sử một giá trị $m$ nào đó, ta kiểm tra xem có tồn tại một đoạn liên tiếp đủ dài để sau khi sửa không quá $m$ cột đèn trong đoạn đó thì sẽ thu được ít nhất $k$ cột đèn còn nguyên vẹn hay không. Việc kiểm tra này có thể được thực hiện bằng cách khai thác các vị trí của các cột đèn bị đổ và xét khoảng cách giữa chúng.

Việc áp dụng thuật toán này ta coi như bài tập.

**Độ phức tạp:** $O(t \log t)$

## Bài 4: Truyền tin

### Tóm tắt đề bài

Cho mảng $h$ gồm $n$ phần tử nguyên dương biểu thị các tần số của các trạm phát sóng. Trong đó, có một thông điệp cần được gửi từ trạm 1 đến trạm $n$. Quá trình truyền tin diễn ra như sau:

- Nếu thông điệp đang ở trạm $i$, nó có thể được phát trực tiếp đến các trạm kề cận nằm trong tầm phát sóng. Cụ thể, thông điệp có thể được truyền đến trạm $i + 1, i + 2, \dots, i + k$.
- Chi phí năng lượng tiêu hao để truyền thông điệp từ trạm $i$ sang trạm $j$ là $|h_i - h_j|$.

Hãy tìm lượng năng lượng tiêu hao nhỏ nhất để đưa thông điệp từ trạm 1 đến trạm $n$.

#### Ràng buộc:

- $2 \le n \le 10^5$
- $1 \le k \le 100$
- $1 \le h_i \le 10^4 \ \forall i \in \{1, 2, \dots, n\}$

#### Subtasks:

- **Subtask 1 (60\%):** $2 \le n \le 100, 1 \le k \le 10$
- **Subtask 2 (40\%):** Không có ràng buộc gì thêm.

### Lời giải

Ta sẽ dùng kỹ thuật quy hoạch động để giải bài toán này. Cụ thể:

- Gọi $f_i$ là lượng năng lượng tiêu hao nhỏ nhất để đưa thông điệp đến trạm $i$.
- Ban đầu, thông điệp đang ở trạm 1, nên ta có: $f_1 = 0$.
- Để đến trạm $i$ ta có thể gửi thông điệp từ các trạm $i - j$, $\forall j
  \begin{cases}
  1 \le j \le k \\
  1 \le i - j
  \end{cases}$
- Khi đó, ta có công thức tính $f_i$ như sau:
  $$
  f_i = \min_{1 \le j \le k, 1 \le i - j} \{f_{i - j} + |h_i - h_{i - j}|\}
  $$
- Vì kết thúc ở trạm $n$ nên kết quả của bài toán là $f_n$.

**Độ phức tạp:** $O(n \cdot k)$

<details>
<summary>Code tham khảo</summary>

```cpp

*    author:  ilovedeque
 *    created: 24.04.2026 14:30:17
**/

#include <bits/stdc++.h>
#define ll long long
#define int128 __int128
#define ld long double
#define st string

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);
    if (fopen("SIGNAL.inp", "r")) {
        freopen("SIGNAL.inp", "r", stdin);
        freopen("SIGNAL.out", "w", stdout);
    }
    int n, k;
    cin >> n >> k;
    vector <ll> h(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> h[i];
    }
    vector <ll> dp(n + 1, 1e18);
    dp[1] = 0;
    for (int i = 2; i <= n; i++) {
        for (int j = max(1, i - k); j < i; j++) {
            dp[i] = min(dp[i], dp[j] + llabs(h[j] - h[i]));
        }
    }
    cout << dp[n];
    return 0;
}
```

## Bài 5: Tập xe

### Tóm tắt đề bài

Cho mảng $A$ gồm $n$ phần tử và số nguyên dương $m$. Hãy đếm số cặp $(i, j)$ $(1 \le i < j \le n)$ sao cho $A_i + A_j \le m$.

#### Ràng buộc:

- $1 \le n, m \le 10^6$
- $1 \le A_i \le 10^6 \ \forall i \in \{1,2, \dots, n\}$

#### Subtasks:

- **Subtask 1 (40\%):** $n, m \le 10^3$
- **Subtask 2 (30\%):** $n, m \le 10^5$
- **Subtask 3 (30\%):** Không có ràng buộc gì thêm.

### Lời giải

#### Subtask 1:

Ta chỉ cần duyệt qua mọi chỉ số $(i, j)$ và đếm số cặp thỏa mãn yêu cầu.

**Độ phức tạp:** $O(n^2)$

#### Subtask 2, 3:

Ta hoàn toàn có thể sắp xếp lại mảng $A$ theo chiều tăng dần.

<details>
<summary>Chứng minh.</summary>

Gọi $B$ là mảng thu được sau khi đã sắp xếp mảng $A$. Khi đó, mảng $B$ sẽ là một hoán vị của mảng $A$, tức là tất cả phần tử trong mảng $B$ xuất hiện cùng số lần với các phần tử trong mảng $A$.

Suy ra, khi ta tìm được một cặp $(i, j)$ thỏa mãn $B_i + B_j \le m$ thì sẽ luôn luôn tồn tại được đúng hai giá trị thỏa mãn đó trong mảng $A$ và ngược lại. Vì vậy, với mọi cặp thỏa mãn trong mảng $B$ thì sẽ có một cặp tương ứng thỏa mãn trong mảng $A$ và ngược lại. Vì thế mà ta có thể sắp xếp mảng $A$ mà không thay đổi kết quả.
</details>

Khi đó, ta sẽ dùng kỹ thuật hai con trỏ để tính đáp án. Cụ thể:

- Ta sẽ khởi tạo biến $i = 0, j = n, ans = 0$.
- Trong khi $i < j$, ta sẽ kiểm tra điều kiện $A_i + A_j \le m$.
- Nếu điều kiện trên được thỏa mãn, ta sẽ cập nhật $ans \rightarrow ans + (j - i)$ và $i \rightarrow i + 1$.
- Ngược lại, ta sẽ cập nhật $j \rightarrow j - 1$.
- Sau khi đã lặp hết (tức là $i \ge j$), đáp án của bài toán là $ans$.

Bây giờ ta đi chứng minh vì sao thuật toán này đúng.

<details>
<summary>Chứng minh.</summary>

Vì mảng $A$ đã được sắp xếp nên ta có: $A_1 \le A_2 \le \dots \le A_n$.

Giả sử ta tìm được cặp $(i, j)$ thỏa mãn $A_i + A_j \le m$, mà ta có $A_k \le A_j \ \forall \ i < k \le j$.

Suy ra, $A_i + A_k \le m \ \forall \ i < k \le j$. Vì vậy, tất cả các cặp $(i, k)$ sao cho $i < k \le j$ sẽ thỏa mãn yêu cầu đề bài. Có thể dễ dàng chứng minh được là có $j - i$ cặp thỏa mãn. Từ đó, ta sẽ cập nhật $i \rightarrow i + 1$ vì ta đã xét hết các cặp thỏa mãn của chỉ số $i$ rồi.

Ngược lại, giả sử $A_i + A_j > m$, mà ta có $A_k \ge A_j \ \forall \ k \ge j$.

Suy ra, $A_i + A_j > m \ \forall \ j \le k \le n$. Vì vậy, tất cả các cặp $(i, k)$ sao cho $j \le k \le n$ sẽ không thỏa mãn yêu cầu đề bài. Do đó, ta có thể cập nhật $j \rightarrow j - 1$ để giảm tổng $A_i + A_j$.
</details>

**Độ phức tạp:** $O(n \log n)$

<details>
<summary>Code tham khảo</summary>

```cpp
/**
*    author:  ilovedeque
 *    created: 24.04.2026 15:22:36
**/

#include <bits/stdc++.h>
#define ll long long
#define int128 __int128
#define ld long double
#define st string

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
```
